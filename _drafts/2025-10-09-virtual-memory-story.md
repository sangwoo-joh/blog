모든 프로그램은 필연적으로 메모리를 사용해야 한다. 현대의 소프트웨어는 모두 폰 노이만 아키텍쳐 위에서 동작하기 때문에 데이터(와 코드)를 담을 공간이 필요하기 때문이다. 

이로 인해서 모든 소프트웨어는 메모리 계층 구조의 한계를 벗어날 수 없다. 결국 계산은 CPU에서 이루어져야 하기 때문에, 물리적으로 CPU에 가까울 수록 메모리 속도가 빨라진다. 그래서 대부분의 현대 컴퓨터는 다음과 같은 메모리 계층 구조를 가진다.

```
CPU (register) -> L1 Cache -> L2 (-> L3) -> RAM -> SSD or HDD
```

 * **레지스터**: CPU 내부에 있어서 가장 속도가 빠르다. 데이터 뿐만 아니라 실행할 명령어(기계어)도 직접 처리할 수 있다. 대신 용량이 매우 작아서 수백 바이트 수준이다. 속도는 거의 1 싸이클 미만이다.
 * **L1 캐시**: CPU 코어에 내장되어 있고 캐시 중에서 가장 빠르다. 보통 명령어를 위한 캐시(Instruction Cache)와 데이터를 위한 캐시(Data Cache)로 분리되는 경우가 많다. 대충 32-128KB 정도의 크기를 가진다. 속도는 러프하게 4 싸이클 정도라고 보면 된다.
 * **L2 캐시**: L1보다 용량은 크지만 좀 느린 캐시다. 역시 CPU 코어 근처에 있다. 용량은 256K-1MB 정도이고 속도는 10-20 싸이클 정도.
 * **L3 캐시**: 요건 선택사항이다. 멀티 코어 프로세서가 도입되면서 등장한 캐시로 여러 개의 코어가 공유하는 캐시다. 수십 MB 정도의 용량과 50-80 싸이클 정도의 속도를 가진다.
 * **RAM**: 여기부터는 이제 우리에게 익숙한 "메모리"다. RAM은 Random Access Memory의 줄임말이다. Random Access는 주소값만 알면 순차적인 접근 없이 데이터를 읽고 쓰고 할 수 있다는 뜻이다. 대충 요즘은 최소 2기가에서 수십, 수백GB까지 다양하게 선택할 수 있다. 속도는 놀랍게도 100-300 싸이클 정도 된다. 프로그램이 데이터를 "쓰는(write)" 최종 목적지 중 하나다.
  * **저장장치**: SSD, HDD 등으로 전원이 꺼져도 데이터를 유지한다(비휘발성). 용량은 가격에 따라 수십, 수백기가에서 수테라까지 다양하다. 속도는 SSD냐 HDD냐에 따라서도 크게 다르지만 보통 RAM보다 수백, 수천, 수만배는 느리다. 프로그램이 데이터를 쓰는 최종 목적지 중 하나다.

모든 계산에 필요한 코드와 데이터는 레지스터까지 와야 한다. 하지만 프로그램이 필요로 하는 데이터와 코드는 메모리에 있다. 캐시들은 레지스터와 메모리 사이에서 지역성(Locality)에 근거해서 프로그램이 더 빨리 실행되도록 돕는 보조적인 역할을 한다.

그런데 프로그램은 저마다 필요로 하는 메모리의 크기가 다르다. 어떤 애는 수 MB의 메모리만으로도 충분하지만, 코드를 컴파일하거나 커다란 데이터를 분석하는 등의 작업에는 수십, 수백, 수테라의 메모리가 필요할 때도 있다. 하지만 물리적인 메모리(RAM)의 용량은 한계가 있고 현대의 운영체제는 이런 프로그램들을 모두 동시에 실행하도록 해야 한다. 물리적인 메모리를 잘 쪼개서 프로그램마다 할당한다고 해도, 어떤 프로그램이 얼만큼의 메모리가 필요한지를 항상 알아낼 수는 없기 때문에 (정지 문제), 이는 꽤 골치아프다.

**가상 메모리**는 이런 한계를 극복하기 위해, 저장장치인 SSD/HDD를 마치 RAM처럼 활용하는 메모리 관리 기법이다. 각 프로세스는 자기 만의 독립적인 (가상) 메모리 공간을 가지는데, 이는 물리적인 메모리와는 분리되어 있다. 실제 물리적인 메모리에는 한계가 있을지 몰라도 그것과 별개로 가상의 메모리 공간을 빌려 준 다음, 프로그램이 실제로 물리적인 한계 이상을 쓰게 되면, 넘친 부분을 저장장치로 대체해서 물리적인 RAM 크기보다도 더 큰 메모리를 쓸 수 있게 해준다.

현대의 가상 메모리 기법은 소프트웨어와 하드웨어 모두의 도움을 통해 구현된다. 소프트웨어 적으로는 페이징 기법이나 관련 알고리즘 등이 있고, 하드웨어 적으로는 MMU(Memory Management Unit; 가상 주소를 물리 주소로 변환하는 걸 도와줌)이나 TLB(Translation Lookaside Buffer; MMU를 위한 특수한 캐시) 등이 있겠다.
